보안과 할루시네이션

Copilot을 켜기 전에는 자료 입력 가능성을 판단하고, 결과를 받은 뒤에는 사실성과 출처를 확인합니다.

목차이전 장다음 장

세 단계 판단법

구분예시처리
입력 가능공개 자료, 더미 데이터그대로 실습 가능
가공 후 가능이름을 지운 회의 메모식별 가능성 재확인
입력 금지개인정보, 계약서 원문, 인사자료사용하지 않음
회사 계정이라고 모든 자료를 넣어도 되는 것은 아닙니다. 조직 정책과 데이터 등급이 우선입니다.

수강생 질문 대응

수강생: 회사 계정이면 안전한 거 아닌가요?
강사: 조직용 보호가 적용될 수 있지만, 회사 계정이라는 이유만으로 모든 자료 입력이 허용되는 것은 아닙니다.
수강생: 고객명을 A사로 바꾸면 괜찮나요?
강사: 좋은 시작입니다. 다만 금액, 일정, 특수 조건으로 고객을 유추할 수 있다면 더 가공해야 합니다.

할루시네이션 대처법

AI가 자신 있게 틀리는 현상을 할루시네이션이라고 부릅니다. 수업에서는 “AI가 거짓말을 한다”보다 “출처 없이 그럴듯한 문장을 만들 수 있다”라고 설명하면 수강생이 방어적으로 받아들이지 않습니다.

아래 답변을 검토해줘.
1. 원문 자료에 근거가 있는 문장과 없는 문장을 구분해줘.
2. 근거가 없는 문장은 삭제하거나 [추가 확인 필요]로 표시해줘.
3. 숫자, 날짜, 고유명사는 별도 검증 목록으로 뽑아줘.
  • 숫자, 날짜, 사람 이름, 계약 조건은 반드시 원문과 대조합니다.
  • “아마”, “일반적으로”, “가능성이 높다” 같은 표현은 근거를 다시 요구합니다.
  • 외부 배포 문서는 Copilot 결과가 아니라 사람이 승인한 문서로 취급해야 합니다.
